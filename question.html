Question-2: What is the difference between innerHTML, innerText, and textContent ?


==>1.innerHTML: * Retrieves or sets the HTML content inside an element. * Parses HTML tags and renders them. * Can be used to insert HTML dynamically.
2.innerText: * Retrieves or sets the visible text inside an element. * Ignores hidden elements (like display: none). * Does not parse HTML tags; it treats them as plain text. * Reads text as it is displayed in the browser.
3. textCotent: * Retrieves or sets the entire text content of an element, including hidden text. * Does not parse HTML tags; it treats them as plain text. * Faster than innerText because it does not check for visibility.


Question-3: What is event delegation in the DOM?


==> Event delegation is a technique in JavaScript where instead of adding event listeners to multiple child elements, you add a single event listener to a parent element. The event then "bubbles up" from the child to the parent, allowing you to handle events more efficiently.


question-4: What is event bubbling in the DOM?


==> Event bubbling is a concept in the DOM event model where an event starts from the target element and then propagates upward through its ancestors (parent elements) until it reaches the document object.
